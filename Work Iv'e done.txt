Run:

hexdump -C tankdata.bin -s 3 | grep -o "[A-Z][A-Z][A-Z][A-Z]\." --color=yes | sort | uniq | lesser

	FIND ALL UNIQUE four letter (all capitals only!) "words" with a dot after them.
		- NOTE the dot is VERY IMPORTANT. Because hexdump treats ANY non-printable character as a dot!
			- So we don't have to check "grep for AAAA + [any nonprintable]" because the non-printable 
			is already done for us by hexdump. Simpler.
			
		- Note originally I did "FOUR CAPS" and nothing else. BUT, then we get AAAAAAAAAAAAZZAAAA.
			- And how can we tell that apart?
			- HOWEVER, since start with a SIZE value afterward, we can "almost" assume that we're not going to get a letter again after the fourth. It shouldn't be DATAA. Because A would indicate a HUGE number since it's the biggest (IIRC) digit of the number. Which would be a HUGE chunk for a file that more-often-than-not is going to have chunks that aren't MEGS in size.
			- This WILL fail however if we need a four letter chunk followed by a PRINTABLE character in terms of size. Perhaps not A, but something printable but lower in binary. But hopefully this isn't the case.
			- ALSO, this won't catch 'AAA ' (space for last character)
